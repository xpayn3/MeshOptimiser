Format compatibility

Format Open Export Hierarchy Materials Vertex colors Notes
STEP / STP●—●●—Native CAD; routed through local Python (OpenCascade).
GLB●●●PBR●Best all-round. Recommended for Cinema 4D 2026.
GLTF●●●PBR●JSON variant of GLB. Same capabilities, larger.
FBX●●●●—Maya, Blender, Unreal, 3ds Max. Binary 7.4 default; ASCII fallback.
USDZ—●●●—Apple Quick Look, Reality Composer, C4D R26+.
OBJ●●flatvia .mtl—Universal but no transforms; hierarchy baked into names.
3MF●—●●—3D-printing standard. Read-only for now.
STL●●single mesh——Geometry only. 3D printing.
PLY—●single mesh—●Geometry + per-vertex colors. Research / scanning.
CSV—●n/an/an/aParts list / BOM. No geometry written.
Hierarchy — whether the format preserves the assembly tree (groups / nested transforms) on round-trip. Materials — color + shading info per part. PBR means full metalness/roughness; plain ● is diffuse only. Vertex colors — per-vertex RGB baked into the geometry (no separate material).

This page needs a local server

You opened the file directly with file://. Modern browsers block ES modules and WASM in that mode.

Or just double-click start.bat. Then open http://localhost:4242.
